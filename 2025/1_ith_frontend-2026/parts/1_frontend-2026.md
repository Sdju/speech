# AI-конструкторы UI — первый слой разработки

<v-clicks>

- Агенты собирают скелет по фигме/issue, разработчик — ревьюер и «ручной режим»
- Дизайн-системы + токены закрывают 80% «пиксель-перфекта»
- Ручное кодинг-время уходит на сложные сценарии и интеграции

</v-clicks>

# Server-first по умолчанию

<v-clicks>

- RSC/Actions-подобные API есть в React/Vue/Angular/Solid
- Edge-функции и persistent-connection API сближают FE и BFF
- SPA без гидрации толстым бандлом становится редкостью

# Signals + resumability > классический VDOM

<v-clicks>

- Qwik/Solid-подходы просачиваются в мейнстрим-фреймворки
- Холодный старт и TTI улучшаются за счёт ленивой активации
- Хуки и эффекты остаются, но сигналы становятся первым классом

# Edge как runtime, а не просто CDN

<v-clicks>

- Персонализация и AB-тесты выполняются на PoP, не в браузере
- Feature-flags, ABR и кеширование зависят от контекста пользователя
- «Лямбды на границе» становятся частью стандартного деплоя

</v-clicks>

# WebGPU из демо в прод

<v-clicks>

- Визуализации, видеофильтры и 3D идут в браузер без нативных обвязок
- On-device ML (распознавание, генерация) экономит серверный бюджет
- Появляются готовые UI-компоненты с WebGPU внутри (графики, карты, CAD)

</v-clicks>

# WASM/WASI — норма для тяжёлых SDK

<v-clicks>

- Видео/аудио редакторы, CAD, игры и ML-ядра пакуются в wasm
- Бэкенд-модули для валидации/рендеринга переиспользуются между FE/BE
- WASI упрощает работу с FS/сетью, снижая разрыв с native

# Дизайн-токены как контракт

<v-clicks>

- Продукты синхронизируются через токены и primitives, не через форки UI-библиотек
- Темизация и мультибренд закрываются пайплайном токенов
- Фронт/бэк/контент разделяют единый словарь семантических токенов

# Безопасность цепочки поставки

<v-clicks>

- SBOM, подпись артефактов и проверка целостности при `install` становятся дефолтом
- Политики «только подписанные пакеты» входят в CI/CD
- Dev-депы и генеративные артефакты сканируются так же строго, как прод-код

- Общие примитивы и shell управляют маршрутизацией, темами и фичфлагами
- Границы между командами задают контракты компонентов, а не страницы

# Web ≈ Native

<v-clicks>

- Tauri/Capacitor/React Native Turbo + PWA с пушами и платежами идут в прод
- Web-хуки для системных возможностей (камера, файлы, BLE) расширяются
- «Тонкие» обёртки побеждают тяжёлые бриджи, падает стоимость кроссплатформы

</v-clicks>

# Метаплатформы вместо «чистых» фреймворков

<v-clicks>

- Next/Nuxt/SolidStart/SvelteKit становятся дефолтом — голый React/Vue реже
- Серверные actions, форм-helpers, кеш API — как часть фреймворка, не экосистемы
- Conformance rules (как в Angular) и lint-плейбуки приходят в React/Vue мир

</v-clicks>

layout: center

# Большая «пятёрка» фреймворков

<v-clicks>

- React: RSC/Actions становятся стандартом, React Compiler шипит в прод
- Vue: сигналы/partial hydration приходят в core, Nuxt держит серверный стек
- Angular: standalone + signals завершены, control flow и conformance — норма
- Svelte 5: runes + серверные хуки в SvelteKit, упор на «zero JS» гидрацию
- Solid/эмиссары сигналов: API просачиваются в mainstream и edge-first рендеринг

</v-clicks>

# Турбосборки: Rust/Go → мгновенный девсервер

<v-clicks>

- Vite остаётся стандартом, но Turbopack/RSBuild/rolldown поджимают
- Инкрементальные графы модулей + кеш по контент-хэшу ускоряют HMR
- Bun/Deno/Node converged: единый ESM, test runner и package manager из коробки

</v-clicks>

# Стейт и данные: меньше клиентских стора, больше fetch-кеша

<v-clicks>

- React Query/TanStack Query-подобные подходы вшиваются в фреймворки
- tRPC/gRPC-web и typed fetch заменяют клиентские BFF-обёртки
- Оптимизация: stale-while-revalidate + partial hydration вместо глобальных стораджей

</v-clicks>

# Тесты и качество по умолчанию

<v-clicks>

- Playwright/Vitest встроены в метаплатформы, снапшоты и a11y чекеры — «галочкой»
- Перформанс-баджеты (CLS/LCP/INP) как CI-полиси, нарушил — билд падает
- Генерация сторибуков/демо из компонентов автоматически, меньше ручных примеров

</v-clicks>
